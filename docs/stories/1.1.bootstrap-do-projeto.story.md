## Story

**As a** Lucas (desenvolvedor solo),
**I want** inicializar o projeto Next.js 16 com TypeScript estrito, Tailwind 4, ESLint, Prettier e fazer o primeiro deploy no Vercel,
**so that** exista uma base técnica versionada, lintada e deployada antes de qualquer linha de código de produto.

## Acceptance Criteria

1. Repositório Git inicializado em `~/Projetos/escritos-da-geo` com primeiro commit contendo `README.md` e `.gitignore` apropriado para Next.js.
2. Projeto Next.js 16 com App Router configurado, usando TypeScript estrito (`strict: true`, `noUncheckedIndexedAccess: true`).
3. Tailwind CSS 4 configurado com `tailwind.config.ts` mínimo e `globals.css` importado no `app/layout.tsx`.
4. ESLint configurado com preset `next/core-web-vitals` + `eslint-plugin-jsx-a11y`. Prettier configurado e integrado ao ESLint.
5. Pre-commit hook (via `lefthook` ou `husky`) executa `lint` + `typecheck` + `format` antes de permitir commit.
6. Repositório conectado ao Vercel; deploy automático em branch `main` retorna HTTP 200 na URL provisória `escritos-da-geo.vercel.app`.
7. Página inicial (`app/page.tsx`) renderiza placeholder textual `"Escritos da Geo — em construção"` (sem styling final).
8. `.env.local.example` criado com lista de variáveis previstas (vazias, sem valores reais).
9. `CLAUDE.md` na raiz com instruções iniciais para AI agents (escopo, stack, convenções).
