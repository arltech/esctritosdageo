# Story 1.2 — Setup do Supabase e Migrations Versionadas

## Status

Ready for Review

## Story

**As a** Lucas (desenvolvedor),
**I want** configurar projeto Supabase local via Docker, integrar cliente no Next.js e estabelecer fluxo de migrations versionadas desde o dia 1,
**so that** toda alteração de schema esteja rastreada e haja banco de dados local pronto para receber as próximas stories.

## Acceptance Criteria

1. Projeto Supabase local rodando via Docker (`supabase start`), com `Project URL` e `anon key` documentados em `.env.local`.
2. Pacotes `@supabase/supabase-js` e `@supabase/ssr` instalados.
3. Cliente Server (`lib/supabase/server.ts`), browser (`lib/supabase/browser.ts`) e admin (`lib/supabase/admin.ts`) implementados conforme docs SSR.
4. Pasta `supabase/migrations/` com 3 migrations: `0001_initial_setup.sql`, `0002_create_profiles_table.sql`, `0003_create_texts_table.sql` (conforme Architecture Seção 9).
5. Trigger `handle_new_user` cria registro em `profiles` automaticamente quando novo usuário é criado em `auth.users`.
6. RLS habilitado em `profiles` e `texts` com policies "deny by default" + leitura pública específica.
7. Script `npm run db:migrate`/`db:reset`/`db:types` configurados.
8. Cliente admin usando `service_role_key` apenas server-side, com aviso explícito.
9. Variáveis `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `ALLOWED_EMAIL` configuradas.
10. Endpoint `app/api/health/route.ts` verifica conectividade Supabase e retorna `{ ok: true, checks: {...} }`.
11. Types TypeScript gerados (`supabase gen typescript`) salvos em `lib/supabase/database.types.ts`.
12. Reexports ergonômicos em `lib/types.ts`.

### Decisões técnicas

- **Portas remapeadas (64xxx)**: outro projeto do Lucas (`team-manager-landing`) ocupava 54322. Em vez de parar (intrusivo), remapeei o range. Documentado no `.env.local`.
- **Keys novas (sb*publishable*/sb*secret*)**: Supabase CLI 2.75 usa o novo formato. Mais curtas que JWT antigo, mantém compatibilidade.
- **Service role client com `import 'server-only'`**: garante erro de bundling se alguém tentar importar em CC. Cached singleton para performance.
- **Middleware Supabase obrigatório**: refresh silencioso de tokens via cookies — sem ele, sessão expira mesmo com usuário ativo.
